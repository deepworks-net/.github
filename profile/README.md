# Deepworks

> _Great tasting code, just like mom used to make!_

**Deepworks** is the software-engineering wing of the [Institute of Conceptual Studies](https://tiocs.io). We build open-source tooling, reusable automation, and engineering standards grounded in **Formal Conceptual Models (FCM)** — the idea that infrastructure should be formally defined and composable, not ad-hoc.

---

## What we're about

- **Architectural purity, practical output** — formally-defined systems that still work natively with the tools you already use.
- **Reuse over reinvention** — quieting the DIY voice with standardized, well-crafted building blocks.
- **Consistency** — shared conventions for versioning, changelogs, and documentation across every repo.

## How we build

Every Deepworks repository follows a shared baseline:

- **Semantic versioning** and **conventional commits**
- **Automated changelog** generation and release drafting
- **MkDocs + Material** documentation deployed to GitHub Pages
- The **six-layer FCM architecture** — formal definitions bridged to GitHub-native compatibility
